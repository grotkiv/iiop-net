Project status

Current release

IIOP.NET allows a seamless interoperation between .NET, CORBA and J2EE distributed objects. Components on each platform can act either in client or server role.

CORBA version

IIOP.NET is based on CORBA 2.3.1.
Only the parts of the specification related to interoperability are implemented.

GIOP Protocol

IIOP.NET supports the following protocols:
IIOP/GIOP 1.0, 1.1, 1.2

The following protocol parts are not supported:

  • Message types other than request, reply, fragment
  • Bidirectional GIOP

Supported types

CLS

Most of the CLS types are supported.

The following types are not supported:

  • some kinds of enums, including flags
  • Delegates
  • Events
  • System.IntPtr
  • System.Decimal

IDL

Most of the IDL types are supported, including CORBA value types.

The following types are not supported:

  • Union
  • Array
  • Long double and fixed

More information can be found in the IDL <-> CLS specification.

ORB Compatibility

Interoperation with the follwoing ORBs has been tested:

  • SUN JDK 1.4 (RMI/IIOP)
  • MICO (in progress)

Interoperation with J2EE applications is tested for the following application servers:

  • BEA Weblogic 6.1
  • IBM Websphere 4.0 (in progress)
  • IBM Websphere 5.0 (in progress)
  • JBOSS 3.2.1 (in progress)
